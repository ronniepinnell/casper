# Research & Analysis

Generate and pressure-test ideas, names, designs, and decisions with structured, adversarial thinking.

[← Back to the collection index](../../README.md)

### Skills (5)

| Unit | What it does | When to call | Install |
|---|---|---|---|
| `brand-name-research` | Generate and vet product/brand names end-to-end | Use for naming a product, app, company, library, or project | `./install.sh --only brand-name-research` |
| `design` | Interactive design session. Agent team debates architecture, captures decisions, logs ideas | — | `./install.sh --only design` |
| `idea-generator` | Generate a large, structured batch of product/SaaS/app ideas on demand | Use when you want NEW ideas (not to evaluate existing ones — that's /idea-pipeline) | `./install.sh --only idea-generator` |
| `idea-pipeline` | Run product ideas through a full funnel — capture/generate → grill → market research → evaluate → rank → graduate the winner (name it, pick a stack,… | Use to develop, pressure-test, compare, or pick between product/app/project ideas | `./install.sh --only idea-pipeline` |
| `meeting` | Context-aware meeting with auto-selected domain experts who debate | — | `./install.sh --only meeting` |
